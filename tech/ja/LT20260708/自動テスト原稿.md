# viMarkdown に組み込んだ話

## 目次
- はじめに
  - マークダウンエディタ viMarkdown 開発中
  - 5～7月：開発版、8月アルファ版、10月ベータ版、12月RC, Stable 予定
  - ver 0.4 の目玉のひとつは vi/ex コマンドサポート
  - vi コマンドの挙動が１文字でもずれているとVimおじさんが怒る
- なぜVimおじさんは怒るのか
  - Vimおじさんはカーソル移動時にどのコマンドを使うか考えていない
  - 移動先を認識するだけで、手が勝手に vi コマンドを奏でる
  - ので、カーソル移動が１文字でもズレているのに耐えられない
- 自動テストの必要性
  - vi コマンドは組み合わせが爆発的（operator × motion × count）
    - opetator: c d y \< >
    - motion: h j k l Spc BS w W e E b B 0 ^ $ f F t T , ; 
  - カーソル位置が1文字でもズレると即アウト
  - 手動テストは無理ゲー
  - 局所的な対処による副作用も心配
- テストケース
  - テスト名、初期テキスト・カーソル位置 ＋
  - （vi/ex コマンド, 期待されるテキスト・カーソル位置, ）・・・
  - カーソル位置を ┃、選択範囲を 《、》 で表現

> { "Move cursor left", "h┃ello\n",
>     {"h",┃hello\n", "h", "┃hello\n", } },
> { "Visual mode", "h┃ello\\n",
>     {"v", "《┃e》llo\\n", "l", "h《e┃l》lo\\n", } },

- テストケース作成
  - 基本的に生成AIに作成してもらう
  - Gemini先生が作成したテストケースを、チャッピー先生・Claude 先生がレビュー・修正・追加
  - テスト失敗時もAIに評価させる
    - vi/ex 実装、テストケース、自動テストに誤りがある場合あり
- テスト結果メッセージ
  - 正常時
    - 何をテストしたか、テスト件数表示
  - テスト失敗時
    - Before:（実行前） ➔ Commands: ➔ Expected: ➔ Actual: を表示
- まとめ
  - vi完全互換は想像以上に難しい・コストを要する
  - だが自動テストがあると安心して実装できる
  - Vimおじさんにも怒られにくくなる
  - viMarkdown ダウンロード・試用よろしく～
