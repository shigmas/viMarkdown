- vi/ex 自動テストの必要性
  - vi コマンドは組み合わせが爆発的（operator × motion × count）
    - opetator: c d y \< >
    - motion: h j k l Spc DS w W e E b B 0 ^ $ f F t T , ; 
  - カーソル位置が1文字でもズレると即アウト（怒るVimおじさん）
  - 手動テストは無理ゲー
  - 局所的な対処による副作用も心配
- テストケース
  - テスト名、初期テキスト・カーソル位置 ＋
  - （vi/ex コマンド, 期待されるテキスト・カーソル位置, ）・・・
  - カーソル位置を ┃、選択範囲を 《、》 で表現

> { "Move cursor left", "h┃ello\n",
>     {"h",┃hello\n", "h", "┃hello\n", } },
> { "Visual mode", "h┃ello\\n",
>     {"v", "《┃e》llo\\n", "l", "h《e┃l》lo\\n", } },

- テスト結果メッセージ
  - 正常時
    - 何をテストしたか、テスト件数表示
  - テスト失敗時
    - Before:（実行前） ➔ Commands: ➔ Expected: ➔ Actual: を表示
