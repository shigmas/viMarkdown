# viMarkdown 罫線保護機能実装

## アウトライン

- 罫線機能：罫線モードにし、Ctrl + 上下左右矢印で罫線描画
```keisen
    ┏━━━━━━━━━━━━━━┓
    ┃■ test dialog            Ｘ┃
    ┠──────────────┨
    ┃                            ┃
    ┃      後悔しませんか？      ┃
    ┃                            ┃
    ┃                            ┃
    ┃    【Ｙｅｓ】【Cancel】    ┃
    ┗━━━━━━━━━━━━━━┛
```
- 文字を入れるたびに罫線図形が崩れるのはストレス
```keisen
    ┌────────┐
    │                │
    └────────┘
             │
             │枠内に abc 挿入（罫線保護機能なし）
             ↓
    ┌────────┐
    │abc                │
    └────────┘
```
- 罫線保護機能とは
  - 自動で空白を調整し、罫線位置を維持する
```keisen
    ┌────────┐
    │                │
    └────────┘
             │
             │枠内に abc 挿入（罫線保護機能あり）
             ↓
    ┌────────┐
    │abc             │
    └────────┘
```
  - 上書きモードがあればよいと思う人もいるかもしれないが、
  - 半角空白を全角文字で単純に置き換えると罫線がずれてしまう（その逆も同様）
  - 半角・全角文字入力・削除に対応した処理アルゴリズムが必要
- 罫線保護アルゴリズム
  - QTextDocument::contentsChange(pos, removed, added) シグナル利用
  - ドキュメントの内容が変更されるたびに発行
  - 引数：カーソル位置、削除文字数、挿入文字数
  - 編集位置の右に罫線文字があれば、削除・挿入文字数を参照し、空白を挿入・削除
  - ただし、文字幅は半角・全角で異なるので文字数→文字幅変換が必要
  - 挿入された場合は、編集後テキストを参照すればよい
  - 削除された文字列は残っていない
    - カーソル位置テキストを保持しておく
    - QPlainTextEdit::cursorPositionChanged() 利用
  - IME入力した場合
    - 読み文字入力中：カーソル以降の文字が右にズレ、
    - 変換確定時にカーソル以降の文字がもとに戻される
    - それが contentsChange() に反映される
    - 挿入文字列、削除文字列を参照し、後ろから同一部分を削除
    - 後はIME未使用の場合と同じ処理
