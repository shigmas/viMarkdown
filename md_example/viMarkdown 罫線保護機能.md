# viMarkdown 罫線保護機能実装


- 罫線保護アルゴリズム
  - QTextDocument::contentsChange(pos, removed, added) シグナル利用
  - ドキュメントの内容が変更されるたびに発行
  - 引数：カーソル位置、削除文字数、挿入文字数
  - 編集位置の右に罫線文字があれば、削除・挿入文字数を参照し、空白を挿入・削除
```CPP
void MarkdownEditor::onContentsChange(int pos, int removed, int added) {  // シグナルにコネクトされた関数
  if( pos の右に罫線文字が無い ) return;
  if( removed != 0 ) 罫線の直前にremoved数の空白を挿入;
  if( added != 0 )  罫線の直前の空白をadded数だけ削除;
}
```
  - ただし、文字幅は半角・全角で異なるので文字数→文字幅変換が必要
  - 挿入された場合は、編集後テキストを参照すればよい
  - 削除された文字列は残っていない
    - カーソル位置テキストを保持しておく
    - QPlainTextEdit::cursorPositionChanged() 利用
  - IME入力した場合
    - 読み文字入力中：カーソル以降の文字が右にズレ、
    - 変換確定時にカーソル以降の文字がもとに戻される
    - それが contentsChange() に反映される
    - 挿入文字列、削除文字列を参照し、後ろから同一部分を削除
    - 後はIME未使用の場合と同じ処理

## 罫線保護モード
viMarkdownには、テキストベースで下図のような図解を直感的に作成できる「罫線機能」が搭載されています。  
Shift + F5 により専用の罫線モードに切り替えた後、Ctrlキーと上下左右の矢印キーを組み合わせることで、キャンバスに絵を描くように自由な形状の罫線を描画することが可能です。

```keisen background-color: #e0e0e0;
    ┏━━━━━━━━━━━━━━┓
    ┃■ test dialog            Ｘ┃
    ┠──────────────┨
    ┃                            ┃
    ┃                            ┃
    ┃      後悔しませんか？      ┃
    ┃                            ┃
    ┃                            ┃
    ┃    【Ｙｅｓ】【Cancel】    ┃
    ┗━━━━━━━━━━━━━━┛
```
しかし、こうしたテキスト図形を運用する上で最大の障壁となるのが、罫線枠内に文字を書き込む際の挙動です。一般的なエディタでは、文字を挿入するたびに右側の罫線が押し出されて図形が崩れてしまい、その都度手動で空白を調整し直さなければならないという大きなストレスがありました。
```keisen background-color: #e0e0e0;
    ┌────────┐
    │                │
    └────────┘
             │
             │枠内に abc 挿入（罫線保護機能なし）
             ↓
    ┌────────┐
    │abc                │
    └────────┘
```
この課題を根本から解決するのが、本アプリ独自の「罫線保護機能」です。これはドキュメントの変更をリアルタイムに監視し、テキストの挿入や削除に合わせて背後の空白を自動で調整することで、罫線の位置を常に一定に維持する機能です。

```keisen background-color: #e0e0e0;
    ┌────────┐
    │                │
    └────────┘
             │
             │枠内に abc 挿入（罫線保護機能あり）
             ↓
    ┌────────┐
    │abc             │
    └────────┘
```
「標準的な上書きモードを使えば済むのではないか」という考え方もありますが、日本語環境においてはそれだけでは不十分です。半角空白を全角文字で単純に置き換えると、文字幅の差異によって結局は罫線がずれてしまうからです。

そのため、単なる上書き処理ではなく、半角・全角それぞれの文字入力や削除に細かく対応し、文字幅単位で帳尻を合わせるための専用アルゴリズムが必要不可欠となります。この仕組みによって、viMarkdownは「崩れない罫線アート」を実現し、快適な図解混じりの執筆体験を提供します。
